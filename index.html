Courier Change

A multisignature File Request System

Development of a decentralized file request system , addressing the needs of institution such as Universities

   One problem that people often overlook is inconvenience brought by physical document requests.   
We do not realize that this process involves a degree of trust from the document issuing party, the courier service, and as well as the receiving party .

We also face the risks associated with the delivery process itself such as lost, damage, or delays. Additionally, document delivery services could also cost a fortune.    It would be nice to have a system that uses cryptography to guarantee the access, privacy, and authenticity of a document as well as leverage the benefits brought by Filecoin and IPFS in ensuring the integrity and decentralization of storing a file. And this is the system we intend to build and provide.

Steps on File Request

Step 1:

File receiver would request for a file

Indicating the File issuer and relevant information
Step 2:

Issuer uploads encrypts and upload file

File issuer would make sure only authorized party can access and read the file
Step 3:

File receiver confirms and access file

File request receives approval from all parties concerned and can now decrypt and view the file

Authentication

+

Decentralization

Leveraging the use of decentralized file systems such as IPFS and Blockchain technologies such as Filecoin
+

Enhance Security

Allows trustless escrow file accesses as well as making it substantially more difficult to be compromised
+

Better Customer Experience

Provides faster service by cutting out the latency from traditional courier services
